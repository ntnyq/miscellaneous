汇众联合门户前端开发指南

 

环境搭建

假设你已安装 Node 与 Git。

 

前置要求

  • 已安装包管理器 Yarn
  • 已安装 Docker 并注册了 Docker Hub
  • 已加入 汇众联合 Code 阿里云 项目组，且拥有开发者权限。

 

配置Hosts

配置你电脑的 hosts，可参考 Mac配置Host域名映射，添加以下两条规则：

127.0.0.1 portal.hzlh.com # 汇众联合门户网站

    

    
127.0.0.1 oa.hzlh.com # 汇众联合OA系统

需要导入的文件位于目录 hzlh/hzlh-backend/databases。

导入顺序以及文件作用：

  1. database.sql 创建数据库 huizhonglianhe
  2. execute.sql 创建表
  3. default.sql 插入初始化数据
  4. temp/*.sql 插入新功能模块数据

 

页面开发

页面开发过程中，建议注释 hzlh/hzlh-frontend-portal/src/View/Smarty/Config/smarty.conf 中的 portal_public_oss_url = https://huizhong-united.oss-cn-beijing.aliyuncs.com/portal-new 这行，打开被注释的 portal_public_oss_url =。这样本地 Smarty 模板都会使用本地的前端素材文件而并非OSS文件，方便本地开发与调试。

待开发与自测阶段完成后，将代码推送至 Git 仓库即可。

关于 ES6+

为了优化开发体验，项目引入了 Babel，它会将 ES6+ 语法编译至 ES5，但是考虑到编译后代码的兼容性、可读性以及体积，在本项目内，你只能受限制地使用部分常用的 ES6+ 语法，但你也应该优先使用这些新语法，来提高代码的可读性和避免错误：

  • const 与 let 变量声明符
  • 模板字符串
  • 箭头函数
  • 解构赋值
  • 函数参数默认值
  • 解构赋值默认值
  • 对象属性与方法新语法

示例：

关于代码风格

请遵循 企信云前端规范。

此项目通过 husky 配置了 Git Hooks，通过 lint-staged 来调用 ESLint 进行 JavaScript 代码风格检查和调用 Styelint 进行 SCSS 代码风格检查。

建议使用 VsCode 编辑器和 ESLint 插件，以便在开发中解决语法错误。也可通过执行 yarn lint 进行风格检查，执行 yarn lint:fix 来修复所有可自动修复的语法错误。

若存在不可自动修复的错误，必须 手动修复至无错误才可提交至 Git 仓库，严禁删除或者跳掉 Git Hooks，严禁向Git 仓库推送尚存在代码风格问题的代码。
